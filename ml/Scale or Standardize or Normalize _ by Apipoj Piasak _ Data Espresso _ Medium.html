Scale or Standardize or Normalize

Apipoj Piasak

·

Oct 17, 2020

ในการ Train Machine Learning Model ปัญหาที่เราจะต้องเจอบ่อยๆ คือเรื่องของ การกระจายตัวของ Data ตัวอย่างเช่น ยอดขาย อาจจะมีตั้งแต่หลักร้อย ถึงหลักล้าน

ทีนี้ Algorithm ส่วนใหญ่ในงาน ML นั้น ต้องการข้อมูลที่ราบเรียบครับ โดยหลักการที่เรานิยมใช้ในการ Transform data ก่อนที่จะนำไปใช้ในการ Train ML โดยจะมีอยู่สองแบบคือ Normalize กับ Standardize ซึ่งวันนี้เราจะมาดูกันว่าสองวิธีนี้เหมาะกับการใช้งานใน use case ใดบ้าง

Scale

ตามหลักการแล้วมีหลายวิธี แต่ที่นิยมนำมาใช้งานคือ การ Rescale ข้อมูลให้อยู่ระหว่าง 0 กับ 1 ครับ ซึ่งวิธีการนี้เราจะต้องรู้ Min และ Max นะครับ โดยเราจะใช้สูตรนี้ในการ Scale ข้อมูลนะครับ

เรามาลองนำไปใช้งานดูนะครับ เดี๋ยวใช้ iris_dataset นะครับผม

เรามาดูค่า max — min ในแต่ละ column กัน

ทีนี้ผมดู observation แรก จะเห็นว่า sepal length (cm) อยู่ที่ 5.1 นะครับ เราจะลองคำนวนดูโดยใช้สมการของ Scale ที่บอกไป

ทีนี้ใน scikit-learn มี library ให้เราใช้งานได้เลยคือ MinMaxScaler ครับ สามารถเรียกใช้ได้เลย

จะเห็นว่าได้ค่าเดียวกันกับคำนวนด้วยมือเลย

MinMaxScaler เหมาะมากที่จะใช้อันแรกในการ Scale ก่อนเริ่ม Train model

Standardize

เป็นอีกวิธีหนึ่งที่เรานิยมใช้เหมือนกัน โดยจะเหมาะกับข้อมูลที่อยู่ในรูปแบบระฆังคว่ำ หรือ Normal distribution ครับ โดยใช้สมาการนี้ในการ Scale ครับ

จาก Iris dataset เหมือนเดิมนะครับ

เราก็มาทำการหาค่า Standard deviation กับ Mean กันก่อน ในที่นี้ผมใช้ numpy นะครับ เดี๋ยวท้ายๆ บทความผมจะอธิบายว่าทำไม

คราวนี้เรามาลองหาค่า ด้วยสมการของ Standardize กันนะครับ

และแน่นอน Scikit-learn ก็มีฟังก์ชันให้เราใช้งานด้วยเหมือนกัน คือ StandardScaler

จะเห็นว่าได้ค่าเท่ากันนะครับ สำหรับ obsevation แรก

StandardScaler เหมาะกับข้อมูลที่อยู่ในรูปแบบ Normal distribution

ใช้อะไรดี?

จริงๆ เราจะชอบพูดติดปากเวลาเราจะ Scale ข้อมูลว่า เฮ้ย data เรา normalize รึยัง แต่เอาจริงๆ แล้ว มันคือการ Scale ข้อมูลครับ ให้อยู่ในรูปแบบที่ Machine Learning อ่านค่าในการสร้าง Model ได้ง่ายขึ้น แต่วิธีไหนดีที่สุดนั้น ตอบเลยว่า ขึ้นอยู่กับข้อมูลของคุณเลยครับ

โจทย์ที่เราเจอบ่อยๆ คือ การ train model ข้อมูลลูกค้า โดยจะมีข้อมูล อายุ กับ ยอดขาย ปนอยู่ ทีนี้ อายุเนี่ย ไม่เกิน 100 แน่ๆ แต่ยอดขายมีตั้งแต่ 0 ถึงล้าน วิธีการทำให้ Model ของเราทำงานได้ดีขึ้นก็คือการ Normalize data นี่แหละครับผม

จากประสบการณ์ เราก็ควรเริ่มจากอะไรที่ง่ายๆ ก่อน โดยใช้ MinMaxScaler ดูก่อน แล้วค่อยลองขยับไปใช้ StandardScaler

และถ้าเรามี Outlier เยอะ จะลองใช้ RobustScaler อีกตัวก็ดีครับ

*ข้อมูลเพิ่มเติมที่อธิบายได้ดี มีรูปประกอบ

https://benalexkeen.com/feature-scaling-with-scikit-learn/

หมายเหตุ

ที่ผมเลือกใช้ Numpy หาค่า Std เพราะมันมีเรื่อง Degree of freedom เข้ามาเกี่ยวข้อง (เพิ่มเติม https://towardsdatascience.com/why-computing-standard-deviation-in-pandas-and-numpy-yields-different-results-5b475e02d112)

โดยถ้าผมใช้ std ของ pandas จะทำให้ค่าของ StandardScaler ออกมาไม่ตรงกัน เข้าใจว่าใน Scikit-learn จะใช้ numpy ในการคำนวนหาค่า std ค่าเลยออกมาใกล้เคียงกันมากกว่า
